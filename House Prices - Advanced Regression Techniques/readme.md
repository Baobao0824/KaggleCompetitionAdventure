# House Prices - Advanced Regression Techniques

[房价挑战](https://www.kaggle.com/competitions/house-prices-advanced-regression-techniques)是kaggle的一个入门竞赛之一，我们可以从中了解kaggle的大致流程。

首先这个竞赛中，对于一个房子有79个变量，他影响了房子的价格，最终的目标是：对于测试集中的每一个id，我们需要预测`SalePrice`变量的值。文件中有一个`train.csv`是训练文件，然后一个`test.csv`是测试文件，最后的`sample_submission.csv`是示例提交样本。

那么官方给出的做法是让我们用随机森林来做。首先我们要知道什么是随机森林。通常来讲，我们要判断一个网络流量是不是恶意流量，我们完全可以训练一棵决策树。

```text
        长度 > 100?
         /      \
       是        否
     TCP?      UDP?
    /   \      /  \
  恶意 正常  正常 恶意
```

看上去很合理，很快就能判断出来。但是这面临着两个问题：首先是非常容易过拟合，其次是一旦数据开始变化，我们得到的决策树就会完全不同，因为判断条件太少了。如果多个条件同时判断，有若干个满足的，那么就是恶意流量。这听起来很好理解，那这个就是森林。重点来了，我们的森林为什么要叫做随机森林呢？他有两个随机。

+ 随机抽样数据。假设我们有1000条训练数据，随机森林不会把1000条都给每棵树，可能是每个树都随机抽800条。
+ 随机选择特征。什么叫做特征，其实很好理解，我们的房子就有79种特征。对于决策树来说，我们上来先判断哪个特征呢？普通决策树会比较一遍所有的特征，最后看到什么变量最能分得清，我们就用什么变量。
